Есть такой протокол WebRTC - звонки в т.ч. видео напрямую между браузерами.
Про него вагон всякой информации про анализ производительности/качества и т.д.
Пару штук вышлю. Что предлагается:
1. Собрать стенд - на виртуалках, реальных компах - мне всё равно и запустить между ними связь по этому протоколу - найти клиенты/сервер, в целом всё равно какие, желательно с возможностью отключения шифрования, если таковое имеется
2. Снять сетевую трассу вирешакром в процессе одной или нескольких голосовых/видеосессийи
3. Посмотреть вирешароком - что он видит - разметить для себя трассу, понять где установление соединения/обмен данными и т.д.
4. Написать небольшую прогу, которая например для начала будет получать на вход аналогичную сетевую трассу и сообщать следующиее (пусть для начала хоть в консоль или файл):
WebRTC сессия между IP1-IP2, кодек(и), аудио/видео, продолжительность, объём переданных данных 1-2 и 2-1. Например так.
Я рекомендую для написания этой проги использовать pyshark - это питоновая обёртка над консольной версией вирешарк. https://github.com/KimiNewt/pyshark - довольно удобная штука для знакомства с DPI
5. Возникают вопросы - пиши. Если чтото покажется сильно сложным и непонятным - попробуй разобраться, не получится - пиши.
 Параллельно можно писать небольшой документ - для себя и для нас - что такое вообще WebRTC, как это работает, какие есть инструменты для его анализа, например из встроенных в браузеры.
